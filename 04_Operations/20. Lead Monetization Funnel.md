# DYV INC — Lead Monetization Funnel

Операционная система каскадной монетизации одного лида через несколько банковских продуктов.

- **Версия:** 1.0
- **Дата обновления:** 2026-07-11
- **ГЕО:** RU
- **Текущий бюджет:** 2 000 ₽
- **Доступные источники:** УБТ, сайты знакомств, сарафанное радио, Telegram, социальные сети, партнёры

## 1. Цель системы

Максимизировать не выплату с одного оффера, а подтверждённый доход с одного привлечённого человека.

Главная единица аналитики — лид, прошедший несколько совместимых продуктов:

`Источник → лид → Регбиз + РКО → дополнительные РКО → дебетовые карты → подтверждённый доход`

## 2. Основной каскад

| Этап | Продукт | Ставка | Целевое действие | Холд | Условие перехода |
|---:|---|---:|---|---|---|
| 0 | Квалификация лида | — | Подтверждены интерес, согласие и соответствие условиям | — | Лид понимает последовательность и готов выполнять действия |
| 1 | Озон Регбиз + РКО | 11 000 ₽ | Оборот 3 000 ₽ за 2–3 дня | 30 дней | Заявка принята, счёт открыт, клиенту дано понятное задание по обороту |
| 2 | Альфа РКО | 12 000 ₽ | Открытие счёта | 30 дней | Предыдущая заявка оформлена; следующий продукт не мешает выполнению первого ЦД |
| 3 | Сбер РКО | 4 000 ₽ | Пополнение 1 001 ₽ | В среднем 14 дней | Клиент готов выполнить простое ЦД и не перегружен оформлениями |
| 4 | Другие РКО | По базе офферов | По условиям оффера | По базе офферов | Выбор по потребности клиента, ставке, сложности ЦД и сроку холда |
| 5 | Дебетовые карты | По базе офферов | Покупка или транзакция | По базе офферов | РКО-этап не требует срочного действия либо основные ЦД уже выполнены |

Потенциальная выплата первых трёх банков — **27 000 ₽ с одного лида**. Это не прогноз и не подтверждённый доход: сумма становится фактической только после выполнения ЦД и подтверждения партнёрками.

## 4. Правила работы с лидом

1. Не оформлять следующий продукт без понятного согласия клиента.
2. До отправки заявки проверять актуальность оффера, допустимость источника и требования к новому клиенту.
3. Не отправлять все продукты одновременно: сначала завершать критичные действия текущей волны.
4. После каждого оформления фиксировать следующее действие и контрольную дату.
5. Не считать заявку доходом до подтверждения партнёркой.
6. Не обещать гарантированное одобрение, выплату или результат.
7. Если продукт не подходит клиенту, пропускать его, а не ломать весь каскад.
8. Персональные данные, телефоны, документы и банковские сведения не хранить в GitHub.

## 5. Статусы этапов

| Статус | Значение |
|---|---|
| `Новый` | Лид получен, квалификация не проведена |
| `Квалифицирован` | Интерес и базовое соответствие подтверждены |
| `Заявка` | Заявка отправлена |
| `Оформлен` | Продукт оформлен, ЦД ещё не подтверждено |
| `ЦД выполнено` | Клиент выполнил требуемое действие |
| `Подтверждён` | Партнёрка подтвердила выплату |
| `Отклонён` | Заявка или выплата отклонена |
| `Пауза` | Клиент временно не готов продолжать |
| `Завершён` | Подходящие этапы каскада пройдены |

## 6. Минимальный трекер лида

Рабочий трекер хранится вне GitHub. В базе знаний фиксируется только его структура.

| Поле | Что хранить |
|---|---|
| Lead ID | Анонимный идентификатор, например `L-0001` |
| Источник | УБТ, знакомства, сарафанка, Telegram, соцсети или партнёр |
| Расход | Фактическая стоимость привлечения |
| Текущий этап | Номер и название этапа каскада |
| Статус | Статус текущего продукта |
| Следующее действие | Что должен сделать клиент или менеджер |
| Контрольная дата | Когда проверить выполнение |
| Оформленные продукты | Перечень без персональных банковских данных |
| Ожидаемая выплата | Сумма ещё не подтверждённых ставок |
| Подтверждённый доход | Только принятые партнёркой выплаты |
| Причина отказа | Краткая стандартизированная причина |

## 7. Метрики каскада

| Метрика | Формула |
|---|---|
| CPL | Расход / количество привлечённых лидов |
| Продуктов на лида | Количество оформленных продуктов / количество лидов |
| Подтверждённых продуктов на лида | Подтверждённые продукты / количество лидов |
| Доход на лида | Подтверждённый доход / количество лидов |
| Конверсия этапа | Лиды, перешедшие дальше / лиды, вошедшие в этап |
| ROI каскада | (Подтверждённый доход − расход) / расход × 100% |
| Срок монетизации | Дни от получения лида до последней подтверждённой выплаты |

При нулевом расходе ROI не рассчитывается: фиксируются доход на лида и абсолютная прибыль.

## 8. Текущая бюджетная рамка

Бюджет `2 000 ₽` используется для привлечения лидов в каскад, а не дробится на отдельный тест каждого банковского оффера.

- до `1 500 ₽` — основной тест привлечения;
- `500 ₽` — резерв на корректировку источника, объявления или связки;
- УБТ, сарафанное радио, партнёры, Telegram и соцсети используются параллельно без обязательного расходования резерва;
- масштабирование начинается только после появления подтверждённого дохода, а не по количеству заявок.

## 9. Исходный кейс

| Показатель | Значение |
|---|---|
| Оффер | Альфа Регбиз + РКО |
| Канал | Личный кабинет банка |
| Источник | Друзья / сарафанное радио |
| Ставка | 7 000 ₽ |
| Лиды | 4 |
| Расход | 0 ₽ |
| Максимальная потенциальная выплата | 28 000 ₽ |
| Подтверждённый доход | 28 000 ₽ |
| Результат | 4 из 4 лидов оплачены |

Этот кейс — подтверждённая исходная точка системы: четыре оплаченных лида без расходов на привлечение. ROI при нулевом расходе не рассчитывается; подтверждённый доход на лида составил 7 000 ₽.
